# ProgramViz

ProgramViz 是一个基于 Qt 6 的轻量工程代码结构与代码量可视化工具。它扫描用户选择的工程目录，以文件树为数据基础，用从下到上生长的紧凑树状图展示目录层级和代码规模，帮助开发者快速了解工程的结构、体量和热点区域。

## 项目功能

### 工程扫描

- 支持选择并扫描一个工程根目录。
- 默认扫描工程根目录下的 `src` 目录，以及工程目录下其他源代码文件。
- 递归识别文件夹与源代码文件，保留实际文件树层级。
- 支持配置源代码扩展名、忽略目录和忽略文件模式，默认忽略构建产物、版本控制目录和缓存目录。
- 扫描过程中显示进度；遇到无权限或无法读取的文件时跳过并记录诊断信息，不影响其他内容展示。

### 代码规模统计

- 统计每个文件的代码行数，并向上汇总到所属文件夹和工程节点。
- 区分总行数、有效代码行数、空行和注释行，具体统计口径在设置中明确展示。
- 为文件记录文件大小、创建时间、最后修改时间、扩展名和扫描状态。
- 文件夹节点显示其直接内容与所有后代节点的汇总信息。
- 对符号链接、重复扫描路径和无法识别的文件类型进行安全处理，避免重复计数。

### 树状图可视化

- 使用紧凑的树状图按文件树展示工程结构。
- 布局从左到右生长：左侧为工程根节点，文件夹和文件沿层级向右展开。
- 每个文件或文件夹使用同一视觉体系的圆角矩形表示，不同层级使用同色系深浅区分。
- 同一层级的兄弟节点之间保留间隔，避免节点粘连。
- 同级节点使用固定的紧凑上下间距；父文件夹矩形的高度与右侧子节点堆叠高度对齐。
- 节点默认字号为 16 pt，块高度与代码量成正比，同时不低于文字高度的 2 倍。
- 节点列宽按当前字号和同层最长文件名自适应，优先保证文件名完整显示。
- 代码行数放在名称第二行，并在详情面板中提供精确统计；节点之间使用层级连接线表达父子关系。
- 对超小节点提供最小可见尺寸和滚动区域，防止大量小文件无法操作。
- 支持垂直滚动、适应窗口和重新布局。
- 在工程规模变化时保持布局稳定，减少重新扫描后节点位置的跳变。

### 交互与详情

- 鼠标悬停时轻微放大或提亮节点，并显示当前节点的层级路径。
- 侧边详情面板显示选中节点的信息，包括：
  - 名称和完整路径
  - 节点类型（文件或文件夹）
  - 文件夹大小或文件大小
  - 内含总代码量及代码行数构成
  - 文件扩展名和识别语言
  - 创建日期和上次修改日期
  - 子文件数量、子文件夹数量和扫描状态
- 点击文件夹节点可在 Finder 中打开对应目录。
- 点击文件节点可使用 VS Code 打开文件；若 VS Code 不可用，则给出明确提示并保留复制路径操作。
- 支持右键菜单：在 Finder 中显示、使用 VS Code 打开、复制路径、重新扫描该目录和加入忽略列表。
- 键盘可访问主要节点和详情操作，窗口缩放后信息布局保持可读。

### 外观与显示设置

- 默认使用浅色背景和中饱和度的多色系配色，支持跟随系统、浅色模式和深色模式。
- 支持“同级同色”和“逐级不同色系”两种配色方案；子层级会继承色系并逐级降低饱和度。
- 提供 HSI 调色板，可自定义色相、饱和度和强度。
- 支持设置节点文字字号，主题、配色方案、HSI 参数和字号通过 `QSettings` 持久化保存。
- 节点文字居中显示，文件或文件夹名称位于第一行，代码行数位于第二行。
- 同一文件夹下同名的头文件和源文件可合并成一个节点，使用细线弱分隔，节点只显示一次文件名。
- 合并节点的 `.h` 与源文件分别显示各自代码行数，分别在左右区域内居中对齐。

## 目标用户场景

- 接手陌生项目时快速建立目录和代码规模认知。
- 通过面积占比定位代码量最大的模块和文件。
- 在重构前识别目录层级过深、模块过大或文件异常集中的位置。
- 以文件树和代码量的共同视角进行项目评审、教学和演示。

## 技术方案

- **语言与框架**：C++ 20、Qt 6 Widgets、Qt Core、Qt Gui。
- **构建系统**：CMake 3.21+、Ninja；提供 Debug 和 Release preset。
- **平台优先级**：优先支持 macOS，后续扩展 Windows 和 Linux。
- **核心模块规划**：
  - `Scanner`：目录遍历、文件过滤、文件元数据读取。
  - `LineCounter`：按语言和统一统计口径计算代码行数。
  - `ProjectModel`：以内存树或 Qt Model 表示工程节点及汇总指标。
  - `LayoutEngine`：将树模型转换为从下到上的矩形布局。
  - `VisualizationView`：节点绘制、缩放、平移、悬停和选中交互。
  - `DetailsPanel`：节点详情展示及外部应用打开操作。
  - `Settings`：源代码类型、忽略规则、主题和行为配置。

## 设计约束与统计口径

1. 文件节点默认使用可读的固定列宽；文件名长度较大时自动扩展列宽，空文件仍保留可点击的最小高度。
2. 文件夹代码量为所有后代源代码文件的汇总，不将二进制文件计入代码量。
3. 默认显示有效代码行数，同时保留总行数、空行和注释行供详情面板查看。
4. 文件夹节点在左侧列显示，后代节点在右侧列按代码规模顺序排列，并通过连接线保持层级关系。
5. 节点颜色表达层级与状态，不直接表达代码质量；避免把视觉大小解释为质量评分。
6. 大型工程扫描必须在工作线程执行，UI 线程只负责状态更新和绘制。
7. 外部打开操作使用系统安全的路径参数，不拼接未经处理的 shell 命令。

## 本地构建

首次打开项目时，请确保已安装 Qt 6.8+、CMake 和 Ninja。

```sh
cmake --preset debug
cmake --build --preset debug
open build/debug/ProgramViz.app
```

在 VS Code 中可使用 CMake Tools 的 **Build**、`F7` 和 `F5`，或从 **Tasks: Run Task** 运行 Debug、Release 和清理任务。首次打开目录时如需选择 preset，请选择 `debug`。
